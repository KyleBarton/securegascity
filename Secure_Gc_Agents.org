* Summary

Gas City agents must be hardened against two failure modes: capability
bleed (an agent accessing paths or credentials it shouldn't) and identity
bleed (an agent inheriting the user's personal Claude configuration).
Proper isolation requires three things working together: running every
agent inside a nono sandbox to enforce filesystem boundaries at the OS
level, scoping Claude's config directory to the city rather than ~/, and
supplying secrets through injection rather than ambient environment
inheritance.

* Notes

** CLAUDE_CONFIG_DIR

Claude Code reads its user settings (=settings.json=, =CLAUDE.md=, memory,
credentials, session history) from =~/.claude= by default. Setting the
=CLAUDE_CONFIG_DIR= environment variable redirects all of that to an
arbitrary path.

- Gas City's =passthroughEnv()= (=cmd/gc/cmd_start.go:946=) already
  forwards =CLAUDE_CONFIG_DIR= to every managed agent session, so setting
  it once before =gc start= propagates automatically — no per-agent config
  needed.
- The supervisor service allowlist (=cmd/gc/cmd_supervisor_lifecycle.go:391=)
  also includes =CLAUDE_CONFIG_DIR=, so it survives launchd/systemd restarts.
- Per-city isolation: set =CLAUDE_CONFIG_DIR= to =<city-dir>/.claude= before
  launching =gc=. All agents in that city share one city-scoped identity,
  fully separate from the user's personal =~/.claude=.
- Per-rig isolation is also possible via =agent.env= in =city.toml=, but
  per-city granularity is usually the right level.

** Nono provider config (city.nono.toml)

The file below registers =nono= as a Gas City provider. The intent is that
every agent spawned under this config runs inside a nono sandbox, so the
OS-level capability boundary is enforced automatically rather than relying
on the agent's own discipline.

#+begin_src toml
[workspace]
name = "my-city"

[providers.nono]
display_name = "Nono"
command = "nono"
args = ["run", "--allow", "/path/to/city", "--credential", "anthropic"]
prompt_mode = "arg"                           # or "flag" if nono takes --prompt
ready_delay_ms = 3000
ready_prompt_prefix = "> "                   # whatever nono's shell shows
process_names = ["nono"]
instructions_file = "AGENTS.md"

# If nono supports session resume:
# resume_flag = "--resume"
# resume_style = "flag"

[[agent]]
name = "worker"
provider = "nono"
start_command = "your agent command"
#+end_src

Several fields still need real values once the exact nono invocation is
known: the =--allow= path(s), =prompt_mode=, =ready_prompt_prefix=, and
=start_command=.

*** Nono profiles

nono profiles are JSON files that group credentials and network settings,
eliminating the need to list =--credential= flags individually on every
invocation. The provider spec in =city.nono.toml= references a profile by
name:

#+begin_src toml
args = ["run", "--allow", "/path/to/city", "--network-profile", "my-city-profile"]
#+end_src

**** Where profiles must live

nono only looks for user profiles in =~/.config/nono/profiles/=. Profiles
must be placed there directly — there is no flag to point nono at an
arbitrary path.

By convention, Gas City profiles should be named =gc_<city-name>_<agent-name>.json=:

#+begin_src sh
~/.config/nono/profiles/gc_myproject_worker.json
~/.config/nono/profiles/gc_myproject_reviewer.json
#+end_src

This makes it easy to identify which city and agent a profile belongs to
when multiple cities are in use.

**** Profile structure

A minimal profile for a Gas City agent that needs Anthropic access:

#+begin_src json
{
  "network": {
    "credentials": ["anthropic"]
  }
}
#+end_src

Additional credentials (e.g. =openai=, =github=) can be added to the
=credentials= array as needed. Built-in service routes for all major LLM
providers are pre-configured by nono.

*** Secret injection via nono's Phantom Token Pattern

Plaintext secrets must never appear in the provider =env= block or
anywhere the agent process can read them. nono's recommended mechanism is
the *Phantom Token Pattern*:

1. Before the sandbox is applied, nono starts a localhost proxy that holds
   the real credentials (retrieved from the OS keychain or 1Password).
2. The sandboxed agent receives only a short-lived, cryptographically
   random session token and an SDK-level base URL pointing at the proxy
   (e.g. =ANTHROPIC_BASE_URL=http://127.0.0.1:<port>/anthropic=).
3. The agent makes normal API requests to localhost. The proxy validates
   the session token, strips it, injects the real API key as an HTTP
   header, and forwards the request over TLS to the upstream service.
4. If the agent is compromised, attackers obtain only the session token —
   a 64-character hex string useless outside =127.0.0.1:<port>= and
   expired when the session ends.

The actual values do not need to be managed by =sgc= at all, and no
=[providers.nono.env]= block is needed in =city.nono.toml=. nono's proxy
is not a standalone process — it is built into =nono run= and runs
per-agent invocation. When an agent is launched via the nono provider
spec (=nono run --credential anthropic -- claude ...=), nono's supervisor
starts the proxy internally, generates the session token, and injects
=ANTHROPIC_BASE_URL=, =NONO_PROXY_TOKEN=, and related vars directly into
that agent process. No external proxy management is required.

The =sgc= wrapper therefore only needs to handle =CLAUDE_CONFIG_DIR=
scoping (see Concrete steps). Credential injection is entirely the
responsibility of the provider spec and nono's runtime.

* Concrete steps

1. Finalize =city.nono.toml= with the real nono invocation (=--allow= paths,
   =prompt_mode=, =ready_prompt_prefix=, =start_command=) and commit it.

2. Set =CLAUDE_CONFIG_DIR= to a city-scoped path before launching =gc= so
   agents never inherit =~/.claude=. Add the following shell function to
   your =~/.zshrc= (or equivalent):

#+begin_src sh
sgc() {
  echo -n "City location: "
  read city_path
  CLAUDE_CONFIG_DIR="${city_path}/.claude" gc "$@"
}
#+end_src

   On invocation, =sgc= prompts for the city directory, then runs =gc= with
   =CLAUDE_CONFIG_DIR= scoped to that city. Any arguments passed to =sgc=
   are forwarded to =gc= unchanged (e.g. =sgc start=, =sgc stop=).

3. Ensure secrets are injected explicitly rather than inherited from the
   ambient shell environment. Pass provider credentials via =agent.env= in
   =city.toml= or supply them as targeted environment variables alongside
   =CLAUDE_CONFIG_DIR= in the =sgc= wrapper — do not rely on globally
   exported tokens in the user's shell profile.
